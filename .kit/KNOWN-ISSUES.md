## Severity Levels

- **Critical** — blocks functionality, must fix before launch
- **High** — bad user experience, fix before launch
- **Medium** — suboptimal but works, fix after launch
- **Low** — minor, address when convenient

---

## How to Use

When the agent finds an issue during build:
1. Add a row to Open Issues immediately
2. Note the severity
3. Critical or High: stop and discuss with user before continuing
4. Medium or Low: log and continue, address later

When resolved:
1. Remove from Open Issues
2. Add to Resolved Issues with fix description
3. Log the fix in CHANGELOG.md
